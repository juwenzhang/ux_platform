# Proposal: 企业级主题低代码配置平台

## 概述

将现有的功能性主题切换系统升级为企业级可视化低代码主题配置平台。提供拖拽式画布预览、实时颜色编辑器、主题 token 管理，并支持导出标准化 JSON 格式，满足设计系统团队和前端团队协作的需求。

## 背景与动机

当前项目已具备基础的三套主题切换能力（浅色 / 深色 / 品牌），但存在以下限制：

- **主题创建依赖代码**：新增或修改主题需要开发者手动编写 TypeScript 配置，设计师无法直接参与
- **缺乏可视化预览**：调整颜色后必须在真实应用中才能看到效果，反馈链路长
- **无法导出/导入**：主题配置与代码耦合，无法在团队间流转或跨项目复用
- **不可扩展**：只有 15 个固定的颜色 token，无法适应更复杂的设计系统

需要将其升级为一个 **可视化的主题配置工作台**，让设计师和开发者都能直观地创建、编辑、预览和导出主题。

## 目标

1. **可视化 Token 编辑器** — 通过颜色选择器、滑块等 UI 控件实时编辑主题变量
2. **画布实时预览** — 在独立的预览画布中实时渲染典型 UI 组件，反映当前主题效果
3. **主题管理面板** — 管理多套主题：创建、复制、删除、重命名
4. **JSON 导入/导出** — 支持将主题配置导出为标准化 JSON，也可导入 JSON 快速创建主题
5. **主题对比** — 并排预览两套主题的差异
6. **低代码布局** — 左侧属性面板 + 中间画布 + 右侧主题列表的经典低代码布局

## 非目标

- 不做完整的低代码页面搭建（只做主题配置，不涉及组件拖拽编排）
- 不做后端存储（全部 localStorage + JSON 文件导出）
- 不做主题市场/在线共享
- 不做 CSS-in-JS 或 Tailwind 生成器（专注于 CSS Variables token）

## 成功标准

- 用户可通过 UI 创建全新主题并实时预览效果
- 导出的 JSON 结构清晰，可被其他项目消费
- 画布预览包含至少 8 种常见 UI 组件（按钮、输入框、卡片、标签、表格等）
- 主题编辑操作响应 < 16ms（实时无延迟）
- 整体界面符合主流低代码平台的交互范式
